# 1. JS原始数据类型有哪些？引用数据类型有哪些？

在JS中，存在着6种原始值也叫基本类型，分别是：

  • Boolean
  • Null
  • Undefined
  • Number
  • String
  • Symbol（ES6）
  • BigInt（ES2020)

引用数据类型：对象Object（包含普通对象-Object， 数组对象-Array，正则对象-RegExp，日期对象-Date，数学函数-Math，函数对象-Function）

# 3. Null是对象么？为什么

结论：Null不是对象

解释：虽然typeof null 会输出object，但是这只是一个JS存在的一个悠久bug。在JS的最初版本中使用的是32位操作系统，为了性能考虑使用低位存储变量的类型信息，000开头代表是对象然后null表示为全0，所以将它错误判断为object

# 4. ‘1’.toString()为什么可以调用

其实在这个语句的运行过程中做了这样几件事情：

var s = new String('1');
s.toString();
s = null;
1
2
3

第一步： 创建String类实例。

第二步： 调用实例方法。

第三步： 执行完方法立即销毁这个实例。

整体过程体现了基本包装类型的性质，而基本包装类型恰恰属于基本类型，包括Boolean，Number和String。

参考：《Javascript高级程序设计（第三版）》

# 5. 0.1 + 0.2 为什么不等于0.3？

JavaScript 使用 Number 类型来表示数字（整数或浮点数），遵循 IEEE 754 标准，通过 64 位来表示一个数字（1 + 11 + 52）

  • 1符号位， 0表示正数，1表示负数
  • 11指数位
  • 52位尾数，小数部分（即有效数字）

最大安全数字：Math.pow(2, 53) - 1，转换为整数就是16位，所以0.1 === 0.1，是因为通过toPrecision(16)去有效位之后，两者是相等的。

在两数相加时，会先转换成二进制，0.1 和 0.2 转换成二进制的时候尾数会发生无限循环，然后进行对阶运算，JS引擎对二进制进行截断，所以造成精度丢失。

总结：精度丢失可能出现在进制转换和对阶运算中
